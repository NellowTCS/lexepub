WebAssembly Adapter

LexePub exposes async EPUB parsing to JavaScript and TypeScript through wasm-bindgen.

Publishing and Setup

Build locally using wasm-pack:

# Build the artifact for web invocation
wasm-pack build --target web

API reference

WasmEpubExtractor methods:

  • new()
  • load_from_bytes(uint8Array)
  • get_metadata()
  • get_metadata_json()
  • get_metadata_is_valid()
  • get_chapter_count()
  • get_toc()
  • get_toc_json()
  • resolve_chapter_resource_path(chapterIndex, href)
  • get_resource(path)
  • get_chapter_resource(chapterIndex, href)
  • get_title()
  • get_chapters_text()
  • get_chapters_text_json()
  • get_chapter(index)
  • get_chapter_json(index)
  • get_chapter_text(index)
  • get_total_word_count()
  • get_total_char_count()
  • has_cover()
  • get_cover_image()
  • get_cover_image_len()
  • get_cover_image_format()

Example

import init, { WasmEpubExtractor } from "./pkg/lexepub.js";

async function run(arrayBuffer) {
	await init();

	const ex = new WasmEpubExtractor();
	await ex.load_from_bytes(new Uint8Array(arrayBuffer));

	const metadata = await ex.get_metadata();
	const chapter = await ex.get_chapter(0);
	const textChapters = await ex.get_chapters_text();

	console.log(metadata);
	console.log(chapter);
	console.log(textChapters.length);
}

CSS and AST

  • get_chapter(index) returns parsed chapter data generated by the Rust core.
  • During AST extraction, manifest CSS resources are applied and reflected in element styles fields.

Notes

  • Filesystem-path loading is intentionally not exposed in browser-oriented WASM flows; byte loading is the primary entrypoint.
  • Linked resources inside EPUB archives can be rendered by combining:
    • normalized src/href from AST,
    • resolve_chapter_resource_path(...), and
    • get_chapter_resource(...) or get_resource(...).
